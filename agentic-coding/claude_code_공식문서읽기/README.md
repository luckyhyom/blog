### Claude Code 개요


### CLAUDE.md
언제 작성해야하는가?
1. Claude가 같은 실수를 두 번이상 할 때
2. 코드베이스에 대해 알아야할 정보가 있을 때
3. 지난 세션에서 입력한 것과 같은 수정 또는 설명을 반복할 때
- skill (코드 베이스의 한 부분에만 적용되는 경우)
- 경로 범위 규칙

기타 팁
1. CLAUDE_CODE_NEW_INIT=1을 설정하여 대화형 다단계 흐름을 활성화
2. 크기: CLAUDE.md 파일당 200줄 이하를 목표로 합니다. 더 긴 파일은 더 많은 컨텍스트를 소비하고 준수를 줄입니다. 지침이 커지면 경로 범위 규칙을 사용

3. 일관성: 두 규칙이 서로 모순되면 Claude가 하나를 임의로 선택할 수 있습니다. CLAUDE.md 파일, 하위 디렉토리의 중첩된 CLAUDE.md 파일 및 .claude/rules/을 정기적으로 검토하여 오래되었거나 충돌하는 지침을 제거합니다. 모노레포에서는 claudeMdExcludes를 사용하여 작업과 관련이 없는 다른 팀의 CLAUDE.md 파일을 건너뜁니다.
4. 사용자 수준 규칙은 프로젝트 규칙 전에 로드되어 프로젝트 규칙에 더 높은 우선순위를 제공합니다.
5. paths 필드가 있는 YAML frontmatter를 사용하여 특정 파일로 범위를 지정할 수 있습니다. 
```
your-project/
├── .claude/
│   ├── CLAUDE.md           # 주 프로젝트 지침
│   └── rules/
│       ├── code-style.md   # 코드 스타일 가이드라인
│       ├── testing.md      # 테스트 규칙
│       └── security.md     # 보안 요구사항
```

6. /memory
/memory 명령은 현재 세션에 로드된 모든 CLAUDE.md, CLAUDE.local.md 및 규칙 파일을 나열하고, 자동 메모리를 켜거나 끌 수 있으며, 자동 메모리 폴더를 열 수 있는 링크를 제공합니다. 파일을 선택하여 편집기에서 엽니다.

7. CLAUDE.md 콘텐츠는 시스템 프롬프트의 일부가 아니라 시스템 프롬프트 후 사용자 메시지로 전달됩니다.

8. /compact
/compact 후 Claude는 디스크에서 CLAUDE.md를 다시 읽고 세션에 새로 다시 주입합니다. 하위 디렉토리의 중첩된 CLAUDE.md 파일은 자동으로 다시 주입되지 않습니다.


### Claude Code 모범 사례

1. Claude가 당신을 인터뷰하도록 하기
> 더 큰 기능의 경우 Claude가 먼저 당신을 인터뷰하도록 하십시오. 최소한의 프롬프트로 시작하고 Claude에게 AskUserQuestion 도구를 사용하여 당신을 인터뷰하도록 요청하십시오.

2. 일반적인 실패 패턴 피하기
- 진행중인 작업과 관련 없는 질문은 /btw
- 요청이 명확하지 않아 여러번 요청했을 경우 /clear 하고 다시 명확하게
- CLAUDE.md - 무자비하게 정리하십시오. Claude가 지시사항 없이 이미 올바르게 수행하면 삭제하거나 hook으로 변환하십시오.
